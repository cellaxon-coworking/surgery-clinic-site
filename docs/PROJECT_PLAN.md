# 성형외과 웹사이트 프로젝트 개발 계획

## 1. 프로젝트 개요

### 1.1 프로젝트 목적
성형외과의원을 위한 종합 웹사이트로 병원 홍보, 의학 정보 게시, 고객 상담 기능을 제공합니다.

### 1.2 주요 기능
- **홍보 콘텐츠 관리**: 블로그 게시물, 유튜브 영상 임베드
- **고객 상담 게시판**: 간편가입을 지원하는 공개/비밀 게시판
- **정보 제공**: 진료 과목, 의료진 소개, 시술 상세 정보
- **다국어 지원**: 한국어, 영어, 중국어, 일본어
- **정적 사이트 생성**: DB 편집 후 Worker에 의한 자동 정적 빌드

## 2. 기술 스택

### 2.1 아키텍처 (정적 페이지 생성 + Ranvier 별도 배포)

> **중요**: Ranvier는 Rust + tokio 기반으로 Cloudflare Workers에서 직접 실행할 수 없습니다.
> 별도 서버에 배포하고 Workers에서 API 프록시 형태로 연결합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                            사용자                                   │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│              Cloudflare Pages (SvelteKit 정적 사이트)                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  정적 콘텐츠 (SSG + ISR)                                     │   │
│  │  - /blog/2024/slug.html  (연도별 라우팅)                    │   │
│  │  - /media/2024/video-slug.html                              │   │
│  │  - /[lang]/blog/2024/slug.html  (다국어)                     │   │
│  └─────────────────────────────────────────────────────────────┘   │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│              Cloudflare Workers (Edge Functions)                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  정적 콘텐츠 요청 → R2에서 직접 반환 (캐시)                    │   │
│  └─────────────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  API 요청 → Ranvier 서버로 프록시                            │   │
│  │  - /api/auth/*      → 인증                                   │   │
│  │  - /api/consult/*  → 상담 게시판                             │   │
│  │  - /api/admin/*     → 관리자 기능                            │   │
│  │  - 인증 검증, Rate Limiting, Logging                         │   │
│  └─────────────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  Static Build Worker (선택)                                 │   │
│  │  - Ranvier 웹훅 수신 → 정적 페이지 빌드                       │   │
│  │  - SvelteKit 빌드 → R2 업로드                               │   │
│  └─────────────────────────────────────────────────────────────┘   │
└────────────────────────────┬────────────────────────────────────────┘
                             │ HTTPS (Private)
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│              Ranvier 백엔드 서버 (별도 배포)                         │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  배포 옵션                                                  │   │
│  │  - AWS EC2 / ECS (Seoul Region)                            │   │
│  │  - 국내 클라우드 (NCP, Kakao Cloud, ABLE Cloud)               │   │
│  │  - Docker Container                                         │   │
│  └─────────────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  비즈니스 로직                                              │   │
│  │  - 게시판 CRUD                                              │   │
│  │  - 회원 인증/인가                                           │   │
│  │  - CRM 기능                                                 │   │
│  │  - 웹훅 발송 (DB 변경 시 Worker에 알림)                      │   │
│  └─────────────────────────────────────────────────────────────┘   │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│              데이터베이스 (국내 서버)                                │
│  - AWS RDS (Seoul Region)                                         │
│  - 또는 국내 클라우드 DB (NCP CDB, Kakao Cloud DB)                  │
└─────────────────────────────────────────────────────────────────────┘
```

**왜 별도 서버인가?**

| 제약 | Cloudflare Workers | Ranvier 서버 |
|-----|-------------------|--------------|
| 언어 | JS/TS/WASM | Rust |
| Async Runtime | 내장 (V8) | tokio |
| HTTP 서버 | 내장 Fetch API | hyper |
| DB 연결 | HTTP API만 | 직접 연결 가능 |
| TCP 소켓 | 지원 안 함 | 지원 |

### 2.2 정적 페이지 생성 흐름 (ISR)

```
1. 관리자가 CMS에서 게시물 작성/수정
        ↓
2. DB에 저장 (posts 테이블)
        ↓
3. Ranvier 백엔드가 웹훅 발송
        ↓
4. Cloudflare Worker가 변경 감지
        ↓
5. SvelteKit 빌드 실행 (또는 Edge에서 렌더링)
        ↓
6. 정적 HTML 생성 → Cloudflare R2에 저장
        ↓
7. Cloudflare CDN 캐시 무효화
        ↓
8. 사용자에게 최신 콘텐츠 제공
```

## 6. 정적 페이지 빌드 시스템

### 6.1 빌드 트리거

1. **자동 트리거**: DB 변경 웹훅 수신 시
2. **수동 트리거**: 관리자 페이지에서 요청
3. **예약 트리거**: 매일 정해진 시간

### 6.2 빌드 프로세스

```typescript
// static-worker/src/builder.ts
interface BuildJob {
  type: 'post' | 'media' | 'all';
  targetId?: string;
  language: string;
}

async function buildStaticPage(job: BuildJob) {
  // 1. DB에서 콘텐츠 조회
  const content = await fetchContent(job);

  // 2. SvelteKit으로 렌더링
  const html = await renderPage(content);

  // 3. R2에 업로드
  const path = getStaticPath(content);
  await uploadToR2(path, html);

  // 4. CDN 캐시 무효화
  await purgeCache(path);

  // 5. 알림 발송
  await notifyComplete(job);
}
```

## 8. 보안 고려사항

### 8.1 개인정보보호법 준수
- **데이터 국외 유출 방지**: 고객 정보는 국내 DB에 저장
- **개인정보 수집 동의**: 회원가입 시 동의 절차
- **정보 처리 방침**: 개인정보처리방침 페이지 게시

### 8.2 의료광고 준수
- **의료광고 심의**: 의료광고 심의기준 준수
- **시술 전/후 사진**: 주의사항 표기
- **과장 광고 금지**: 사실에 기반한 정보 제공
- **의료법 준수**: 자세한 내용은 [MEDICAL_COMPLIANCE.md](./MEDICAL_COMPLIANCE.md) 참조

### 8.3 기술적 보안
- **HTTPS**: 모든 통신 SSL 암호화
- **CSRF 보호**: 토큰 기반 검증
- **XSS 방지**: 입력 값 sanitization (DOMPurify)
- **SQL Injection**: Prepared Statement 사용
- **비밀번호 해시**: bcrypt/argon2 사용
- **속도 제한**: API 요청 Rate Limiting
